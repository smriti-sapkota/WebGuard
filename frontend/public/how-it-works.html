How WebGuard Works

A quick look at the scanning flow from start to finish.

Step 1 - Configure

Enter the target URL and optional login credentials.

Step 2 - Authenticate

WebGuard logs in so protected routes can be reached and tested.

Step 3 - Crawl

The crawler discovers pages, forms, and parameters available on the target.

Step 4 - Test

Each input point is tested for SQL Injection, XSS, and CSRF weaknesses.

Step 5 - Report

Findings are saved and shown in the results view with severity and evidence.
